# D&D Campaign Assistant — State

## Status: Complete

## Dev commands
```powershell
# Backend (port 8003)
cd backend
pip install -r requirements.txt
copy .env.example .env    # set AI_PROVIDER=ollama or add ANTHROPIC_API_KEY
uvicorn main:app --reload --port 8003

# Frontend (port 3003)
cd frontend
npm install
npm run dev
```

## Architecture
- **Frontend:** Next.js 14, TypeScript, Tailwind — dark parchment theme (--bg: #0f0e0a)
- **Backend:** FastAPI on port 8003
- **AI:** Ollama by default; supports Anthropic/OpenAI via `providers.py`
- **DB:** SQLite (`dnd_assistant.db`) — auto-created on first run
- **Storage:** `uploads/{campaign_id}/` for gallery images
- **Proxy:** `app/api/backend/[...path]/route.ts` → `127.0.0.1:8003`

## Key features
- Campaign list → per-campaign tabs: Characters, NPCs, Encounters, Session Notes, Lore & Notes, Gallery
- **Characters:** full stat blocks (STR/DEX/CON/INT/WIS/CHA), HP, AC, equipment, spells, notes
- **NPCs:** AI-generated with personality, backstory, secret, motivation, speech pattern
- **Encounters:** AI-generated by difficulty (easy/medium/hard/deadly), environment, party level
- **Session Notes:** single note processing + bulk import (.txt files); Q&A against all session history
- **Lore & Notes:** freeform entries by category (factions, locations, history, religion, etc.)
- **Gallery:** image upload (maps, characters, locations, items) with lightbox view

## Data model
- `campaigns` → `npcs`, `encounters`, `session_notes`, `characters`, `lore_entries`, `images`

## Known issues / lessons learned
- Ollama (llama3.2:3b) sometimes returns list values instead of strings in JSON — backend coerces with `to_str()` helper
- postcss.config.mjs MUST use `export default` not `module.exports` — root cause of original ActionQueueContext crash
- First Next.js compile on Windows takes 2-3 minutes — normal
